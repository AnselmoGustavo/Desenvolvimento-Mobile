## Project Overview

Flora Yamaguti is a local-first desktop/mobile order management app for a small plant business. No backend — all data lives in `localStorage`. Two parallel implementations exist: a React + Electron app and a standalone vanilla HTML file.

## Commands

```bash
npm run dev            # Start Vite dev server (React)
npm start              # Launch Electron app (requires built renderer/)
npm run build          # Build React app into renderer/
npm run electron:build # Build React then launch Electron
npm run package        # Package Windows installer via electron-builder
npm run preview        # Preview production build
```

> For the Android (Capacitor) target, `capacitor.config.json` points web dir at `www/` — copy build output there before running `npx cap sync android`.

## Architecture

### Entry Points

`main.js` loads `renderer/index.html` (saída do build do Vite). Em dev, usa `ELECTRON_START_URL` para apontar para o servidor Vite.

### React App Structure (`src/App.jsx`, ~1,083 lines)

Everything lives in this single file, organized in layers top-to-bottom:

1. **Constants** — localStorage keys (`plt_pedidos`, `plt_fornecedores`, `plt_produtos`, `plt_historico`), status labels, unit options
2. **Utilities** — `formatCurrency()`, `parsePrice()`, `getInitials()`, `genId()`
3. **Data normalization** — `normalizeFornecedores()`, `normalizeProdutos()`, `syncProdutos()` handle legacy data migration and keep supplier↔product references consistent
4. **Hooks** — `useLocalStorage()` (state + localStorage sync), `useAppData()` (all business logic: CRUD for orders, suppliers, products, history)
5. **Design tokens** — `T` (color palette) and `S` (style objects) — all styling is inline via these objects
6. **Small components** — `Badge`, `Chip`, `Btn`, `FormGroup`, `StyledInput`, `Modal`
7. **Screen components** — `PedidosScreen`, `HistoricoScreen`, `FornecedoresScreen`, `ProdutosScreen`
8. **Root `App`** — tab navigation, backup/restore, screen routing

### Data Models

```js
// Pedido (Order)
{ id, cliente, contato, obs, data, status, dataEntrega,
  itens: [{ produto, qtd, fornId, unidade }] }
// status: "pendente" | "em_preparo" | "entregue"

// Fornecedor (Supplier)
{ id, nome, contato,
  produtos: [{ nome, precoEstimado, unidadeTipo, unidadeOutro }] }
// unidadeTipo: "Kg" | "Caixa" | "Vaso" | "Unidade" | "Outro"
```

When a supplier's products change, `syncProdutos()` is called to update all active orders that reference those products.

### Electron Security

`main.js` uses `contextIsolation: true`, `nodeIntegration: false`, `sandbox: true`. `preload.js` is currently a placeholder — do not expose Node APIs there without careful review.

## Key Conventions

- **Currency**: Brazilian Real (BRL), formatted with `pt-BR` locale via `formatCurrency()`.
- **IDs**: Generated by `genId()` (timestamp + random suffix).
- **Styling**: All styles are inline JS objects using the `S` and `T` design token objects — no CSS files, no Tailwind.
- **State**: No state management library. `useAppData()` is the single source of truth, passed down as props.
- **Persistence**: Any new data type needs a new `useLocalStorage` key and a normalization function for forward-compatibility.
